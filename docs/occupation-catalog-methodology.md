# 職業名録・人気アクセス順位の作成方法

## 職業名録

- SSOT: JILPT「職業情報データベース 解説系ダウンロードデータ ver.7.01」
- 取得日: 2026-07-13
- 元データ記載の最終更新日: 2026-04-27
- 収録: 556職業、15分類
- 利用項目: 職業名、別名、主な厚生労働省編職業分類、収録番号、職業概要、仕事内容、就業経路、労働条件、関連資格、関連団体、解説領域の更新年
- 非利用項目: 画像、動画、賃金、求人倍率、能力プロフィール

`scripts/import-jobtag-catalog.mjs` でCSVから上記項目を内部データへ抽出します。公開する `/all/` は、職業名、別名、分類、job tag公式ページへのリンクに限定した単一の検索可能な名録です。job tagの写真・動画は利用しません。

個別 `/occupation/{catalogId}/` の公開状態は `content/occupation-publication.json` を正本にします。レコードがない556職業はすべて `draft` として扱うfail-closed設計です。職業固有の独自本文1,000字以上、公式出典、確認日、関連動画、関連資格、関連AI、執筆者・編集者、人による確認日、承認証跡、公開日が揃った `published` レコードだけが個別ページ、sitemap、llms.txtの詳細導線、`Occupation` JSON-LDへ出ます。現時点の公開済み個別 `/occupation/` は0件です。

人が全主張を確認済みの既存職業解説は `/career/{slug}/` で公開します。`/all/` の該当カードはこの確認済み解説へ内部リンクし、それ以外はjob tag公式ページへ外部リンクします。未確認の職業へ内部リンクを作らないため、名録から404へ進むことはありません。

## 公開・更新ルール

- 名録の元データは、必須4本文（概要・仕事内容・就業経路・労働条件）が欠ける場合、ビルドを失敗させる。ただし元データが揃っているだけでは個別ページを公開しない。
- データ版の更新は自動公開せず、差分を人が確認してから取り込む。
- `content/occupation-publication.json` にない職業は必ずdraftとし、公開条件が1つでも欠ければ個別ページを生成しない。
- 資格名は必須性を意味しない。受験要件と制度変更は実施団体の公式情報で再確認するよう表示する。
- 労働条件は、個別求人の待遇や採用可能性を保証しない。
- `Occupation`構造化データは公開条件を通過した個別ページだけで画面の職業名・概要と一致させ、`JobPosting`、架空の給与、評価、雇用形態は生成しない。

## 人気アクセス順位

順位は厚生労働省job tagの「アクセスランキング」を2026-07-13に目視確認した結果です。アクセス履歴は関心を示す一つの指標であり、年収、将来性、転職しやすさ、適性、採用可能性、職業価値を示しません。

manapickの「動画」「AI」「資格」表示は、その職業から次に確認できる姉妹サイト情報の有無です。この表示を順位計算に使いません。PR報酬、承認状況、EPCも順位に使いません。

## 更新

- job tagデータの版変更時は再取込後に差分を人が確認する。
- アクセス順位は30日ごとに公式ページを再確認する。
- 自動取得結果を直接公開せず、reviewed承認後に反映する。
