# 競合・参照サイト100件の比較記録

確認日: 2026-07-14

## 調査範囲と誠実な表記

`content/competitive-landscape.json` を100件の調査台帳とする。全件で公開トップページまたは公開検索結果から、運営主体と提供領域を確認した。うち `reviewDepth: deep` は、職業詳細、一覧、記事、調査資料など主要画面まで確認したサイトである。`surface` はサイト全体を精読したという意味ではなく、公開画面から分類と主要用途を確認した比較対象である。ログイン必須画面、有料記事本文、robots.txtや利用規約で取得が制限された内容は収集していない。

## 100件から共通して見つかった設計

1. 公的職業データベースは、職業名、タスク、知識・技能、就業経路を広く提供する一方、利用者が「今どの作業を試すか」までつなぐ編集は限定的だった。
2. 民間の職業図鑑は、仕事内容、年収、将来性、資格、転職導線を一ページに集約する傾向がある。ただし、調査対象や確認日がカード上で分かりにくい例、広告・求人導線と編集結論の境界が見えにくい例があった。
3. 求人サービスは検索条件と応募導線が強いが、求人のない職業を含む長期的な探索や、応募前の小さな試作には目的が異なる。
4. 人事・労働調査サイトは一次資料や専門的な解説が充実する一方、職業ページ、学習動画、AI、資格へ横断して戻る導線は少なかった。
5. 学習サービスは開始しやすいが、「受講前に仕事内容を確認する」「習得後にどの仕事で使うか」「価格や報酬と編集順位を分ける」という導線は各サービスの目的外になりやすい。
6. 海外の公的職業サイトは、タスク・技能・職業群を構造化して探索できる。ただし制度、資格、賃金、職業分類は国ごとに異なるため、日本向け本文へ直接移植できない。

## manapick careerで追加・維持する差別化

- 556職業名録の「広さ」と、人が全主張を確認した詳細職業の「深さ」を明確に分ける。
- キャリアデータ室で、指標ごとに「示すこと」「示さないこと」「次に確認すること」を同じ画面に出す。
- 職業名ではなく、入力・作業・成果物・確認責任を先に比較する。
- manapickの動画、manapick AI、manapick licenseを、広告報酬で順位付けせず作業単位で接続する。
- 求人応募や適性判定を行わず、最低三候補と小さな試作へつなぐ。
- 記事は結論先出し、目次、一次出典、確認日、執筆者・編集者、訂正窓口、SNS共有を標準装備する。
- 年収、制度、資格要件など鮮度リスクの高い主張は期限切れをビルドで検出する。
- 広告とPRを本文の結論から分離し、広告枠を主要CTA・比較操作・入口案内の近くへ置かない。

## 追加したコンテンツ

- `/research/`: 公的・一次調査の数値を、転職成功率や職業順位へ変換せず読む「キャリアデータ室」。
- `/news/`: 30本の長文解説。各記事1,000字以上、一次出典、結論、目次、限界、次の行動を必須化。
- `/all/`: 556職業名録と、人が確認済みの詳細職業を区別。
- `/skills/`: スキル名の羅列ではなく、仕事の場面と学び方へ接続。

## 更新方法

競合台帳の件数、URL重複、HTTPS、確認日、調査深度は `npm run competitive:check` で検査する。外部サイトの変更は自動で本文へ反映せず、人が確認してから台帳・設計判断を更新する。100件という件数自体を品質の根拠にせず、具体的に採用した差分を本書へ残す。

2026-07-14のHTTP監査結果は、正常92件、404・410は0件、bot制限（401・403・429）6件、通信エラーによる未確認2件だった。未確認は日本経済新聞の人的資本ページとJobs and Skills Australiaであり、404として扱わずQAに残す。`surface` 75件は主要画面の精読を終えたという意味ではないため、「100サイトを徹底分析済み」とは表記しない。
